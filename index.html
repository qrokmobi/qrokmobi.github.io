SAFETY

CryptoWallet is a gate between external applications and user crypto wallets. During development, the focus was on security and resistance to data substitution.

CryptoWallet doesn't store any password or keys on the hard drive. During compilation, the user should pass two PGP V2 keys — one public — for signature verification, second private — for configuration.

After startup system is needed the special file, that is passed via REST protocol that contains encrypted settings for cryptocurrencies. The application verifies the digital signature and decrypts data to memory. After that this data uses to access to crypto wallets.

After application shutdown or system reboot any sensitive data becomes inaccessible. Also, it is impossible to get sensitive data via any interface.

During every access to cryptocurrencies, system verifies digital signatures.

To access REST interface system should use https + API key, that is set up during dynamic system configuration.

EXPANDABILITY

The application is written using Java 8 and Spring Framework.

Every crypto wallet module is a class that implements the common interface. Modules don't have common memory and interact with other system using task queues.

So to implement new crypto wallet or similar payment system just need to add the new module based on common pattern and recompile the system.

Dynamic adding new modules isn't implemented due to security reasons.

EXPANDABILITY

Access to all functions (sending cryptocurrency, generating new wallet, checking transaction or wallet status, getting balance) implemented via REST API. An external application just needs to pass action and target cryptocurrency.

API calls for all cryptocurrencies are uniform, the diference just in cryptocurrency code. An external application shouldn't take care about internal implementation and specific interacting actions.

REST API is described using SWAGGER V3 (swagger.io) and allows testing and code generation without any real data operation.
